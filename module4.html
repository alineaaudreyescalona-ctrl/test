1. Static Arrays

A static array is a fixed-size data structure that stores elements of the same type in contiguous memory locations. The size must be specified at compile time or initialization and cannot be changed during program execution.

Key Characteristics

Fixed Size

Size determined at creation, cannot grow or shrink

Fast Access

O(1) random access using index

Contiguous Memory

Elements stored in adjacent memory locations

Stack Allocated

Typically stored on the stack (faster but limited)

Declaration Examples

// C/C++

int numbers[5] = {10, 20, 30, 40, 50};
char name[20] = "John";

// Java

int[] numbers = {10, 20, 30, 40, 50};
String[] names = new String[10];

// Python (using array module)

import array
numbers = array.array('i', [10, 20, 30, 40, 50])

Operations & Complexity

Operation Time Complexity Description
Access O(1) Direct index access
Search O(n) Linear search (unsorted)
Insert (end) O(n) Requires shifting elements
Delete O(n) Requires shifting elements

Key Advantage

Static arrays offer predictable memory usage and excellent cache performance due to contiguous memory allocation. Perfect when size is known in advance!

2. Dynamic Arrays

A dynamic array is a resizable array that can grow or shrink during runtime. It automatically handles memory allocation and reallocation, providing flexibility when the size is unknown at compile time.

How Dynamic Arrays Work

Resizing Strategy

  1. 1. Start with initial capacity (e.g., 8 elements)
  2. 2. When full, allocate new array (typically 2x size)
  3. 3. Copy all elements to new array
  4. 4. Delete old array and update reference

Static vs Dynamic Arrays

Static Array

✗ Fixed size
✓ Faster (no reallocation)
✓ Less memory overhead
✗ Can waste space

Dynamic Array

✓ Flexible size
✗ Slower (reallocation cost)
✗ Extra memory for capacity
✓ Efficient memory usage

Examples in Different Languages

C++ (std::vector)

Dynamic
std::vector<int> nums;
nums.push_back(10); // Grows automatically
nums.push_back(20);

Java (ArrayList)

Dynamic
ArrayList<Integer> nums = new ArrayList<>();
nums.add(10); // Grows automatically
nums.add(20);

Python (List)

Dynamic
nums = [] # Dynamic by default
nums.append(10) # Grows automatically
nums.append(20)

Amortized Complexity

Although resizing takes O(n), it happens infrequently. Averaged over many insertions, append operation is O(1) amortized!

3. Multidimensional Arrays

Multidimensional arrays are arrays of arrays, allowing you to store data in multiple dimensions. The most common is the 2D array (matrix), but you can have 3D, 4D, or higher dimensional arrays.

2D Arrays (Matrices)

Declaration

// C/C++
int matrix[3][4];

// Java
int[][] matrix = new int[3][4];

// Python
matrix = [[0]*4 for _ in range(3)]

Common Operations

Traversing (Row-wise)

for (int i = 0; i < rows; i++) {
  for (int j = 0; j < cols; j++) {
    process(matrix[i][j]);
  }
}

Time Complexity: O(rows × cols)

Matrix Addition

for (int i = 0; i < rows; i++) {
  for (int j = 0; j < cols; j++) {
    result[i][j] = A[i][j] + B[i][j];
  }
}

Time Complexity: O(rows × cols)

Memory Layout

In row-major order (C/C++), elements are stored row by row. In column-major order (Fortran), elements are stored column by column. This affects cache performance!

4. String Manipulation

Strings are sequences of characters, typically implemented as arrays. String manipulation involves operations like concatenation, searching, replacing, and parsing - fundamental skills for any programmer.

String Representation

Mutable Strings

Can be modified after creation

Examples:

• C++ (char array)

• Python (list of chars)

Immutable Strings

Cannot be changed once created

Examples:

• Java (String)

• Python (str)

Common String Operations

Concatenation

"Hello" + " " + "World" → "Hello World"

Complexity: O(n + m)

Substring Search

indexOf("World") in "Hello World" → 6

Complexity: O(n × m)

String Reversal

"Hello" → "olleH"

Complexity: O(n)

Split/Tokenize

"a,b,c".split(",") → ["a", "b", "c"]

Complexity: O(n)

String Algorithms

Palindrome Check

bool isPalindrome(str) {
  int l = 0, r = len-1;
  while (l < r) {
    if (str[l++] != str[r--])
      return false;
  }
  return true;
}

Time: O(n), Space: O(1)

Anagram Check

bool isAnagram(s1, s2) {
  if (s1.length != s2.length)
    return false;
  sort(s1); sort(s2);
  return s1 == s2;
}

Time: O(n log n), Space: O(1)

Performance Tip

For many string concatenations, use StringBuilder (Java) or StringBuffer to avoid creating multiple intermediate strings. This reduces time complexity from O(n²) to O(n)!

5. Array Applications

Arrays are used extensively across computer science and software development. Understanding their applications helps you choose the right data structure for your problems.

Real-World Applications

Image Processing

2D arrays store pixel values for images

pixel[row][col] = RGB(255, 0, 0)

Database Tables

Rows and columns stored as 2D arrays

table[row][column]

Signal Processing

Audio/video data stored in arrays

samples[time][channel]

Game Development

Game boards, tile maps, inventories

board[x][y] = piece

Common Array Algorithms

Two Pointer Technique

Pattern

Use two pointers moving from different ends or at different speeds

int left = 0, right = n - 1;
while (left < right) {
  // Process elements
  left++; right--;
}

Use cases: Palindrome check, sorted array problems

Sliding Window

Pattern

Maintain a window of elements and slide it through the array

for (int i = 0; i < n - k + 1; i++) {
  // Process window [i...i+k-1]
  int sum = 0;
  for (int j = i; j < i + k; j++)
    sum += arr[j];
}

Use cases: Maximum subarray sum, substring problems

Prefix Sum

Optimization

Precompute cumulative sums for range query optimization

prefix[0] = arr[0];
for (int i = 1; i < n; i++)
  prefix[i] = prefix[i-1] + arr[i];
// Range sum [l,r] = prefix[r] - prefix[l-1]

Use cases: Range sum queries, subarray problems

Common Pitfall

Array index out of bounds! Always check: 0 ≤ index < array.length. This is one of the most common programming errors!
